# MVP Test Plan - Unified AI Evaluation Platform

## 1. Introduction

This document outlines the test plan for verifying the Minimum Viable Product (MVP) functionality of the Unified AI Evaluation Platform. The primary focus of the MVP is the end-to-end evaluation of Automatic Speech Recognition (ASR) services using a mock vendor adapter.

This plan covers:
*   Admin authentication and authorization.
*   Management of ASR vendor configurations.
*   Management of ASR test cases (audio files and metadata).
*   Creation, execution, and viewing of ASR evaluation jobs and their results.

## 2. Test Environment

*   **Setup:** Follow the instructions in the `docs/MVP_Setup_Guide.md` to set up the local development environment, including Docker (PostgreSQL, MinIO), backend, and frontend.
*   **Admin Credentials:**
    *   Username: `admin` (as per `ADMIN_USERNAME` in `.env`)
    *   Password: `adminmvp` (or as set in `ADMIN_PASSWORD` in `.env`)
*   **Browser:** A modern web browser (e.g., Chrome, Firefox, Safari, Edge).
*   **Test Data:**
    *   **Audio Files:** Prepare 2-3 short sample audio files (e.g., WAV, MP3 format).
        *   `sample_clear.wav`: Clear speech.
        *   `sample_noisy.wav`: Speech with some background noise.
        *   `sample_short.wav`: A very short utterance.
    *   **Ground Truth Transcriptions:**
        *   For `sample_clear.wav`: "The quick brown fox jumps over the lazy dog."
        *   For `sample_noisy.wav`: "Hello world this is a test with noise."
        *   For `sample_short.wav`: "Okay."
    *   **Sample JSON for Parameters (Optional for Job Creation):**
        *   `{"model_preference": "accuracy", "language_hints": ["common_phrases"]}`
